# 명당자리 — 최종 산출물 가이드

## 완성된 결과물

### 1. 로우파이 와이어프레임 (완성)
**파일**: `명당자리 와이어프레임.html`
- **16개 화면** 모두 포함
- 손그림 톤 + 마스코트 자리
- 캔버스: 드래그 정렬, 전체화면 비교 가능
- 섹션: 시작 플로우 / 분석 결과 3가지 변형 / 메인 5탭 / 공유 카드

### 2. 디자인 시스템 (완성)
**파일**: `DESIGN_SYSTEM.md`
- **색상 팔레트**: 오행 기반 (火·水·木·金·土) + 보라 액센트
- **타이포그래피**: Pretendard (본문) + Gamja Flower (제목)
- **컴포넌트**: 버튼, 칩, 카드, 마스코트 스펙
- **간격 & 레이아웃**: 8px 그리드 기반
- **상태 & 애니메이션**: 호버, 로딩, 전환 효과

### 3. 하이파이 화면 HTML (진행 중)
**파일**: `hifi-screens.html`
- 온보딩, 홈, 분석 결과, 명당, 공유 카드 렌더링
- 디자인 시스템 적용 (색상, 타입, 간격)
- 마스코트 SVG 포함
- 브라우저에서 실시간 확인 가능

| 화면 | 목적 | 핵심 요소 |
|------|------|---------|
| **온보딩 1** | 감각적 초대 | 마스코트 + 공간 오브제 |
| **온보딩 2** | 약속 안내 | "공간 가이드" 메시지 |

| **명당** | 지역 추천 | TOP 3 + 침대 방향 + 쇼핑 CTA |
| **마이** | 프로필 & 저장 | 미션 기록, 메뉴 |
| **공유 카드 A** | 귀여운 기본 | 마스코트 + 개운법 |

| **공유 카드 C** | 명당 추천 | TOP 3 지역 명시 |

## 사용 방법

### 와이어프레임 검토
1. `명당자리 와이어프레임.html` 열기
2. 캔버스에서 폰 드래그해 보기, 더블클릭으로 전체화면 모드
3. 피드백 반영 시 해당 화면의 React 파일 (`screens-*.jsx`) 수정

### 디자인 시스템 적용
1. `DESIGN_SYSTEM.md` 읽기
2. 색상, 타입, 간격값 확인
3. 하이파이 작업할 때 이 스펙 준수

### 하이파이 진행
1. `hifi-screens.html` 브라우저에서 확인
2. 필요시 HTML/CSS 직접 수정
3. 최종 PNG로 내보내기 (브라우저 인쇄 > PDF > PNG)

## 주요 디자인 결정사항

✅ **분석 결과**: 카드 스와이프 (A형) 확정
✅ **명당 탭**: 추천 1/2/3 + 부적합 사유 + 쇼핑 CTA
✅ **공유 카드**: A형(귀여움) + 혼합형(귀여움+정보) 병행
✅ **온보딩**: 참고용 텍스트 최소화
✅ **마스코트**: 귀여운 고양이 + 핑크 강조
✅ **색상 시스템**: 오행 (火·水·木·金·土) 팔레트
