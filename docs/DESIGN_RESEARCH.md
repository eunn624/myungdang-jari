## 1. 레퍼런스에서 가져올 점

### 한 화면에 한 메시지

레퍼런스 온보딩은 화면을 가득 채우는 여백 위에 작은 일러스트와 큰 질문 하나만 둔다. 사용자는 복잡한 설명을 읽지 않고 다음 행동으로 자연스럽게 넘어간다.

명당자리 적용:

- 첫 화면 문구를 짧게: “내 사주에 맞는 공간을 볼까요?”
- 부가 설명은 한 줄 이하: “생년월일시는 브라우저에서만 계산돼요.”
- 입력 전 설명을 줄이고, 결과 화면에서 근거를 단계적으로 공개한다.

### 스토리형 결과 공개

레퍼런스의 오늘 운세 화면은 프로그레스 바, 어두운 배경, 중앙 집중형 타이포로 결과를 한 장씩 넘기는 느낌을 만든다.

명당자리 적용:

- 결과를 긴 보고서가 아니라 5장 카드 흐름으로 구성한다.
- `오행 요약 → 부족 오행 → 명당 후보 → 침대 방향 → 오늘의 공간 미션`
- 각 카드에는 한 문장 결론과 근거 칩 2~3개만 노출한다.

### 감정적 진입, 실용적 마무리

레퍼런스는 캐릭터와 말투로 가볍게 시작하지만, 상세 화면에서는 긴 설명을 제공한다.

명당자리 적용:

- 진입부는 부드럽게: “오늘 내 공간에 필요한 기운”
- 본문은 실용적으로: “水 보완: 수변·블루 계열·북쪽 배치 참고”
- 효능 보장 표현은 피하고, “참고”, “가볍게”, “시도해볼 수 있어요” 톤을 유지한다.

## 2. 명당자리 디자인 방향

### 포지셔닝

**캐릭터 운세 앱이 아니라, 사주 기반 공간 큐레이션 매거진.**

무속적인 상징이나 과한 점술 표현은 줄이고, 자연물·집·방향·동네의 감각을 전면에 둔다.

### 비주얼 키워드

- 조용한 공간
- 물길, 나무, 돌, 빛
- 손으로 기록한 듯한 부드러운 리딩
- 매거진형 카드
- 결과보다 “근거”를 신뢰감 있게 보여주는 UI

### 추천 무드

현재 앱은 베이지/블랙 기반으로 차분하지만 조금 도구적이다. 다음 개편에서는 아래처럼 화면별 온도를 나누는 것이 좋다.

| 화면 | 배경 | 핵심 느낌 |
|---|---|---|
| 온보딩 | warm ivory | 조용한 초대 |
| 입력 | warm ivory + thin line | 간단한 의식 |
| 분석 로딩 | deep ink | 몰입감 |
| 결과 요약 | deep ink + element color | 리딩 카드 |
| 명당 추천 | paper white | 근거 있는 큐레이션 |
| 데일리 미션 | pale element tint | 반복 방문 |

## 3. 화면별 제안

### A. 온보딩

현재: 제품 설명 + 입력 폼이 한 화면에 바로 등장  
개선: 레퍼런스처럼 감정적 진입 화면을 먼저 둔다.

추천 구조:

1. 자연물 일러스트 또는 생성 이미지
2. “내 사주에 맞는 공간을 볼까요?”
3. “동네, 침대 방향, 오늘의 작은 배치를 참고해요.”
4. CTA: “시작하기”

주의:

- 고양이 캐릭터는 그대로 가져오지 않는다.
- 명당자리는 방, 창가, 식물, 물결, 지도 조각 같은 공간 오브젝트가 더 잘 맞는다.

### B. 생년월일시 입력

레퍼런스의 큰 질문 + 두꺼운 입력 박스 구조는 좋다.

추천 구성:

- 질문: “태어난 시간을 알려주세요”
- 입력:
  - 양력/음력 segmented control
  - 생년월일
  - 출생 시간
  - “시 모름” 체크
- 하단 고지: “입력값은 브라우저에서만 계산됩니다.”

### C. 분석 로딩

분석 중 화면은 현재 버튼 안의 작은 점 애니메이션보다 별도 화면이 더 좋다.

추천 문구:

- “오행의 균형을 읽는 중”
- “지명과 지형을 맞춰보는 중”
- “침대 방향 가이드를 정리하는 중”

이 로딩은 실제 분석이 빠르더라도 700~1200ms 정도 유지하면 결과가 더 “계산된 것”처럼 느껴진다.

### D. 결과 요약

현재는 결과가 한 화면에 이어서 나와 정보량이 많다. 레퍼런스처럼 첫 결과는 강한 한 장으로 분리한다.

추천 카드:

```text
오늘의 공간 키워드
水 보완

당신의 사주는 火가 강하고 水가 비어 있어요.
공간에서는 물길, 북쪽, 블루 계열을 가볍게 참고해볼 수 있어요.
```

하단 칩:

- 火 62.5%
- 水 0%
- 길방 북
- 침대 방향 남

### E. 명당 후보

행정동 추천은 랭킹처럼 보이면 “효능 보장”처럼 읽힐 수 있다. 점수 대신 “참고 카드”로 유지한다.

추천 카드 구성:

- 행정동명
- 위치: 서울 · 용산구
- 한자: 漢江路洞
- 근거 칩: 水 보완 / 수변 / 한강 인접
- 한 줄 설명: “水 보완 기준에서 지명과 수변 지형을 함께 참고할 수 있어요.”

### F. 침대 방향

나침반 UI는 제거된 스펙이므로 만들지 않는다. 대신 방 안에서 쉽게 상상되는 카드가 좋다.

추천 구성:

- 큰 방향 텍스트: “남쪽”
- 작은 평면도/침대 아이콘
- 대안 팁:
  - “방 구조상 어렵다면 남쪽 벽면을 정리하거나 따뜻한 조명을 두는 식으로 참고해보세요.”

### G. 데일리 루틴

레퍼런스 홈의 핵심은 재방문 이유다. 명당자리도 MVP 이후 바로 이 화면이 필요하다.

추천 카드:

- 오늘의 오행 컬러
- 오늘의 작은 배치
- 오늘 피하면 좋은 과잉 요소
- 추천 소품 1개

예:

```text
오늘의 작은 배치
북쪽에 투명한 유리컵이나 블루 오브제를 두어보세요.
```

## 4. 콘텐츠 톤 가이드

- 이 동네가 당신에게 최고입니다

## 6. 우선순위

### 1순위: 결과 화면 리디자인

가장 제품 가치가 드러나는 곳이다. 현재 기능을 유지하면서도 체감 완성도가 크게 오른다.

작업:

- 결과 요약을 딥 카드로 분리
- 오행 비중을 스토리 카드화
- 명당 후보 카드를 근거 중심으로 정리
- 침대 방향 카드를 더 시각적으로 구성

### 2순위: 온보딩/입력 개선

유입 첫인상을 정리한다.

작업:

- 첫 화면 CTA 추가
- 입력 폼을 단계형으로 변경
- “시 모름” 설명을 더 자연스럽게 정리

### 3순위: 데일리 루틴 홈

리텐션의 핵심이지만, 결과 화면이 먼저 안정된 뒤 붙이는 것이 좋다.

작업:

- 오늘의 공간 미션
- 오늘의 오행 컬러
- 최근 분석 다시 보기
