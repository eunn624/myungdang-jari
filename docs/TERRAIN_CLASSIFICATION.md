# 지형 특성 분류 기준

명당자리 MVP는 행정동 단위로 지형 특성을 분류한다. 정밀 GIS나 사면향 분석은 Phase 1 범위에서 제외하며, 사전 정리된 행정동 속성을 조회하는 방식으로 사용한다.

## 분류 체계

| 코드 | 표시명 | 기준 | 오행 선호 |
|---|---|---|---|
| `highland` | 고지대 | 산자락, 구릉, 경사지, 고지대 주거지 | 土 |
| `waterfront` | 수변 | 강, 천, 호수, 바다, 항만, 계곡 인접 | 水 |
| `flatland` | 평지 | 생활 동선이 비교적 평탄한 도심·평지 | 火·金 |
| `green` | 녹지 | 공원, 산림, 국립공원, 숲길 접근성 | 木 |

## 운영 원칙

- 행정동은 `terrain` 대표값 1개를 반드시 가진다.
- `terrainNote`에 보조 특성이 있으면 런타임에서 `terrainTags`로 함께 판정한다.
  - 예: “한강변, 국사봉 기슭” → 대표값 `waterfront`, 보조 태그 `highland`
- 추천 점수는 대표 지형 일치 시 더 높고, 보조 태그 일치 시 낮은 가중치를 준다.
- 결과 문구는 점수/순위가 아니라 “참고 카드”와 근거 문장으로 제공한다.

## 데이터 확장 로드맵

현재 `data/districts.json`은 수도권(서울·경기) 전체 행정동/읍면동 자동 생성본이다.

1. Phase 1: 서울·경기 전체 행정동/읍면동 목록 자동 수집
2. Phase 1.5: 기존 수작업 정제본과 병합해 대표 지형/오행 속성 자동 부여
3. Phase 2: 하천 거리·공원 면적·표고 데이터로 보정

현재 데이터는 행정안전부 주민등록 인구통계 CSV를 기준으로 생성하며, 스크립트는 [scripts/build_capital_districts.py](/Users/gimsieun/Desktop/02%20실험실/05%20명당자리/scripts/build_capital_districts.py:1)에 있다.
기존 수작업 정제 항목은 우선 보존하고, 신규 항목은 이름 키워드 기반 지형/오행 자동 분류를 적용한다. 추천 품질 검증 후 GIS 보정 작업으로 이어간다.

## 오행별 기본 선호 지형

| 부족/용신 오행 | 선호 지형 | 안내 톤 |
|---|---|---|
| 木 | 녹지 | 공원·숲길·산림 접근성이 있는 동네를 참고 |
| 火 | 평지 | 볕, 활동성, 이동 동선이 편한 평지형 동네를 참고 |
| 土 | 고지대 | 산자락·구릉·안정감 있는 지형을 참고 |
| 金 | 평지 | 정돈된 도심 평지와 생활 인프라 밀집 지역을 참고 |
| 水 | 수변 | 강·천·호수·바다 인접 지역을 참고 |

## 한계

- 이 분류는 MVP 수작업 정제본이다.
- 효능을 보장하지 않으며, 오락·참고 목적의 라이프스타일 가이드로만 사용한다.
- Phase 2에서 공원 면적, 하천 거리, 표고 데이터 등으로 보정할 수 있다.
